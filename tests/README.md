### V2 개선 내용 요약 (DAY8~9)

**고위험 키워드 티어 (highRiskPatterns + highRiskBonus):**

| 신호 유형 | 고위험 KB 키워드 | 기본 가중치 | 고위험 가중치 |
|-----------|-----------------|-------------|---------------|
| pressure | 영장, 체포, 구속, 계정정지, 압류, 벌금 | 16 | 16+30=46 |
| url | bit.ly, tinyurl, han.gl, me2.kr 등 단축URL | 18 | 18+30=48 |
| tooGood | 원금보장, 수익보장, 고수익, 확정수익 | 12 | 12+15=27 |

**콤보 보너스 (V1 유지 + V2 신규 3종):**

| 콤보 | 보너스 | 대표 패턴 |
|------|--------|-----------|
| url + money | +10 | 링크 + 금전 요구 |
| impersonation + personalInfo | +12 | 기관 사칭 + 개인정보 탈취 |
| money + pressure | +8 | 가족 사칭 송금 압박 |
| offPlatform + tooGood | +12 | 카톡 리딩방 |
| impersonation + pressure | +8 | 공문·영장 협박 |

**설계 원칙:**
- 임계값(45)·기존 단일 가중치 체계 유지 (정상 문자 오탐 방지)
- 특정 케이스 텍스트 하드코딩 금지 — 키워드 일반화(티어/콤보)만 적용
- 고위험 KB는 RAG식 구조(로컬 배열)로 관리 — 신고 사례 확인 시 배열에 추가하는 방식으로 확장 가능
- 보너스 값은 라운드 수(10, 12, 15, 28→30 등)로 일반화 — 특정 케이스 점수 정밀 맞춤 배제
