# ADR-0002: 규칙 V2 + Workers AI LLM 하이브리드 채택 (V3)

> 16일 완주 후 · AI 사기 메시지 방패 팀 · 담당: 박서준(개발) · 최유나(평가/데이터)

---

## 상태

- 채택

---

## 맥락 (어떤 상황에서 결정이 필요했나)

- ADR-0001(DAY 9)에서 LLM 외부 호출을 보류하고 규칙 기반 V2로 16일 완주했습니다. 그 결과 Pass Rate 90%(18/20)를 달성했습니다.
- 그러나 ADR-0001에서 명시한 한계 2건이 16일 완주 이후에도 그대로 남아 있었습니다.
  - **case-h01 (지인 사칭 사회공학)**: "링크 눌러봐"처럼 명시적 키워드가 없는 메시지를 규칙 엔진이 탐지하지 못해 정상으로 오판합니다.
  - **case-h02 (드라마 대화)**: "경찰·영장·체포"가 포함된 일상 대화를 규칙 엔진이 맥락 없이 사기로 오판합니다.
- 두 케이스는 모두 **문장의 전후 맥락을 이해해야** 올바르게 판단할 수 있는 구조적 문제입니다. 규칙 기반 엔진만으로는 해결이 불가능합니다.
- 완주 후 Cloudflare Workers AI를 검토한 결과, 서버 바인딩 방식으로 API 키가 불필요하고 추가 비용 없이 LLM을 호출할 수 있음을 확인했습니다. ADR-0001에서 보류한 LLM 비용·키 관리 문제가 해소됩니다.

---

## 결정 (무엇을 선택했나)

- 규칙 엔진 V2(클라이언트 측 실행)와 Cloudflare Workers AI LLM(서버 측 문맥 판단)을 함께 실행하는 **하이브리드 방식**을 채택합니다.
- LLM 모델: `@cf/meta/llama-3.3-70b-instruct-fp8-fast` (Cloudflare Workers AI 서버 바인딩, API 키 불필요).
- 워커 엔드포인트: `POST /api/scam-check` (공개).
- **하이브리드 판정 로직**: 규칙 엔진과 LLM을 함께 실행한 뒤, 두 판정이 일치하면 그 라벨을 채택하고, 불일치하면 LLM(문맥 판단)을 신뢰합니다.

---

## 이유 (왜 이 선택인가)

- **비용·키 부담 없음**: Cloudflare Workers AI는 서버 바인딩 방식으로 동작합니다. 외부 LLM API처럼 월정 비용이나 API 키 관리가 필요 없습니다. ADR-0001 보류 이유였던 비용 제약이 해소됩니다.
- **맥락 한계 2건 해결**: LLM이 문장 전체 맥락을 이해하므로, case-h01(지인 사칭)과 case-h02(드라마 대화)를 올바르게 판단할 수 있습니다.
- **규칙 엔진의 설명 가능성 유지**: 하이브리드 방식에서도 규칙 엔진이 탐지한 신호 목록(위험 신호·점수)은 그대로 사용자에게 보여줍니다. "왜 위험한가"를 설명하는 능력은 유지됩니다.
- **실측 개선**: 하이브리드 Pass Rate 95%(19/20). V2 대비 +5%p.

---

## 고려했지만 택하지 않은 대안

**① 외부 LLM API (GPT-4o, Claude API 등 직접 호출)**

- 강점: 성능이 높고, 프롬프트 제어가 자유롭습니다.
- 미채택 이유:
  - 월정 API 비용이 발생합니다.
  - API 키 노출 방지·관리가 별도로 필요합니다.
  - 응답 지연이 발생할 수 있습니다.
- 결론: Workers AI 서버 바인딩이 비용·키 없이 동일한 목적을 달성할 수 있으므로 미채택.

**② Fine-tuning**

- 강점: 우리 도메인(한국 사기 문자)에 특화된 모델을 만들 수 있습니다.
- 미채택 이유:
  - 충분한 라벨 데이터(수백~수천 건)가 필요합니다. 현재 평가셋 20건은 학습 데이터로 사용하기에 부족합니다.
  - 학습 실행과 결과 검증에 수일 이상이 필요합니다.
- 결론: 차기 과제로 보존.

**③ 규칙 엔진 단독 (V2 동결)**

- 강점: 구조가 단순하고 외부 의존성이 없습니다.
- 미채택 이유: case-h01·h02처럼 맥락 이해가 필요한 케이스를 구조적으로 해결할 수 없습니다.

---

## 결과 / 영향

**Pass Rate (20케이스 기준)**

| 버전 | Pass Rate | 태그별 |
|------|-----------|--------|
| V1 (규칙) | 75%(12/16) | Basic 100% / Edge 60% / Safety 60% |
| V2 (규칙 개선) | 90%(18/20) | Basic 100% / Edge 83.3% / Safety 83.3% |
| **V3 하이브리드** | **95%(19/20)** | **Basic 8/8 / Edge 5/6 / Safety 6/6** |

**해결된 한계**

- **case-h01 (지인 사칭)**: 규칙 엔진 미탐(0점, 정상 오판) → LLM이 문맥으로 **사기** 판단 → 해결.
- **case-h02 (드라마 대화)**: 규칙 엔진 오탐(74점, 사기 오판) → LLM이 일상 대화로 **정상** 판단 → 해결.

**신규 한계 (정직히 기록)**

- **case-e04 (부동산 중개 수수료 이체 — 정상)**: LLM이 "수수료 명목 계좌 요구"를 보수적으로 사기로 판단했습니다. 정상 케이스를 사기로 오탐하는 결과입니다. 사기 방패는 안전 도구이므로 "놓치기보다 한 번 더 의심"하는 보수적 오류는 수용 가능한 범위로 판단했습니다. 다만 AI도 완벽하지 않다는 사실을 정직히 기록합니다.

**가드레일 변화**

- V3에서 사용자 입력이 LLM(Cloudflare Workers AI)으로 전달됩니다. 따라서 기존에 선제적으로 구현해 둔 `detectPromptInjection`(프롬프트 인젝션 탐지)이 **실질적인 방어 역할**을 가지게 됩니다. ADR-0001의 "V2 LLM 도입 대비 선제 구현"이 실제 효력을 발휘합니다.
- PII(개인식별정보)는 Cloudflare Workers AI 내부에서만 처리됩니다. 외부 제3자 API로는 전송되지 않습니다.
