## 채점 방식 (Pass Rate, 라벨 + 신호 부분집합)

### Pass Rate

전체 케이스 중 정확히 분류한 케이스의 비율입니다.

```
Pass Rate = 맞힌 케이스 수 ÷ 전체 케이스 수 × 100(%)
```

V1 규칙 엔진은 다음 두 조건을 모두 만족해야 해당 케이스를 통과한 것으로 봅니다.

1. **라벨 일치**: 엔진이 출력한 "사기/정상" 판정이 기대 라벨과 같아야 합니다.
2. **신호 부분집합**: 엔진이 탐지한 신호 목록이 기대 신호를 모두 포함해야 합니다(기대 신호가 없는 정상 케이스는 라벨 일치만 확인).

### 임계값

- 위험도 점수 **45점 이상** → "사기" 판정
- 위험도 점수 **44점 이하** → "정상" 판정

### 목표 기준

- 03-success.md 연동 성공 기준: **Pass Rate ≥ 85%**
- 최종 확인: DAY 15 Eval

## 실패 4건 분석

공통 원인: **단독 신호만 있어 임계값 45점에 미달 → 사기를 "정상"으로 오판**

| ID | 유형 | 신호 | 점수 | 실패 원인 상세 |
|----|------|------|------|----------------|
| case-e01 | Edge | pressure | 16점 | "당장 연락 안 하면 영장" — 압박 표현 하나만 있어 점수가 16점에 그침. 영장을 앞세운 보이스피싱의 전형적 수법이나 V1은 탐지 불가. |
| case-e03 | Edge | url | 18점 | "bit.ly/abc123" — 단축 URL 하나만 있어 점수 18점. 링크 클릭을 유도하는 사기의 핵심 수단이지만 다른 신호가 없어 미탐. |
| case-s02 | Safety | offPlatform, tooGood | 24점 | 카카오톡 리딩방 유도 + 원금보장 100% 수익 — 두 신호 합산 24점. 전형적인 투자 사기임에도 복합 신호가 임계값 미달. |
| case-s03 | Safety | pressure, money | 38점 | "엄마야, 당장 이 계좌로 보내줘" — 압박 + 금전 요구 합산 38점. 임계값 7점 미달로 미탐. 피해 금액이 크고 빈도도 높은 수법. |

### 분석 요약

4건 모두 **단독 또는 소수 신호 조합이 임계값을 넘지 못하는 구조적 한계**에서 비롯됩니다.

- V1 규칙 엔진은 신호가 여러 개 겹쳐야(콤보) 위험 점수가 45점을 넘기 쉬운 구조입니다.
- 반면 실제 고위험 사기 수법은 단 하나의 신호(강압적 언어, 단축 URL)만으로도 피해를 유발합니다.
- 규칙을 하나씩 땜질해 임계값을 낮추면 이 4건은 통과하지만, 정상 문자를 사기로 잘못 잡는 오탐(false positive)이 증가합니다(과적합 위험).

## 개선 방향 (DAY8 모델 · DAY9 RAG로 단독 신호 약점 보강)

V1 기준선 75%를 정직하게 박제하고, 개선은 DAY8~9 구조적 접근으로 진행합니다.

| DAY | 접근 | 기대 효과 |
|-----|------|-----------|
| **DAY 8** | LLM 도입 — 문맥 이해 능력 추가 | "당장 연락 안 하면 영장"처럼 표현만으로 압박 의도를 파악. 단독 신호도 문맥 안에서 고위험으로 분류 가능. |
| **DAY 9** | RAG 도입 — 최신 사기 수법 사례를 참고 자료로 활용 | 리딩방 유도·가족 사칭 송금처럼 새로운 수법을 사례 DB에서 검색해 보완. 규칙에 없는 패턴도 탐지 가능. |

실패 4건(case-e01·e03·s02·s03)은 DAY8·DAY9 개선 작업의 근거 데이터셋으로 활용합니다.
