# MVP 성공 기준 (03-success.md)

> DAY 3 · AI 사기 메시지 방패 팀 · 작성자: 이도윤 (PM)

## 성공 기준 한 문장

> 테스터가 의심 문자의 사기 여부를 기존 평균 30초에서 5초 안에 판단하고, 라벨된 평가셋 Pass Rate ≥ 85%를 달성한다.

## 측정 방법

- **무엇을 측정하는가:**
  - ① 사기/정상 분류 정확도 (Pass Rate): 서비스가 사기 메시지를 사기로, 정상 메시지를 정상으로 맞히는 비율.
  - ② 판단 소요 시간: 테스터가 문자를 붙여넣고 "사기다/아니다" 결론을 내리기까지 걸리는 시간. 서비스 사용 전(Before)과 후(After)를 비교한다.

- **어떻게 측정하는가:**
  - Pass Rate: DAY 7에 설계한 라벨된 테스트셋(사기·정상 혼합, 최소 16개 케이스)을 서비스에 입력해 정답 라벨과 대조한다. 맞힌 케이스 수 ÷ 전체 케이스 수 × 100 = Pass Rate(%).
  - 판단 소요 시간: 초기 테스터(DAY 4 인터뷰 대상자)에게 서비스 사용 전·후 각각 의심 문자 5개를 보여주고 결론을 말하는 데 걸리는 시간을 스톱워치로 측정한다.

- **목표 수치:**
  - Pass Rate ≥ 85% (1차 목표. V2(LLM 개선) 단계에서 90% 도전)
  - 판단 소요 시간 ≤ 5초 (기존 평균 약 30초 대비)

## 성공/실패 판단 시점

- **1차 판단**: DAY 15 최종 Eval — 라벨된 평가셋 전체를 돌려 Pass Rate를 최종 확인한다.
- **재검증**: DAY 15 최종 Eval과 같은 날, DAY 4에 참여했던 초기 테스터가 서비스를 다시 사용해 판단 시간 Before/After를 비교한다.
- 두 조건(Pass Rate ≥ 85% AND 판단시간 ≤ 5초)을 모두 충족해야 성공으로 기록한다.
