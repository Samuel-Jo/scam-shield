# 방식 비교 리포트 — V1 vs V2 (DAY8)

> DAY 8 · AI 사기 메시지 방패 팀 · 담당: 박서준(개발) · 최유나(평가/데이터)

---

## 비교 대상

우리 프로젝트에서 '모델 라우팅'은 두 AI 모델을 갈아 끼우는 것이 아니라, **같은 규칙 엔진을 어떻게 구성하느냐**의 두 가지 방식을 비교하는 것으로 해석했습니다. 조그만 규칙을 조금씩 고치는 '땜질'이 아니라, 신호 강도를 체계적으로 분류하는 '구조 개선'이 실제 모델 성능 차이를 만드는지 숫자로 검증합니다.

| 방식 | 설명 |
|------|------|
| **A = V1 (기본 규칙)** | 신호별 단일 가중치. 임계값 45점 미만이면 아무리 위험한 단독 신호여도 정상 판정. |
| **B = V2 (고위험 티어 + 콤보 보너스)** | 단독 신호라도 고위험 키워드(KB)가 포함되면 추가 점수 부여. 신호 조합(콤보)에도 추가 점수. 임계값·기존 단일 가중치는 동결. |

---

## 채점 방식

평가셋과 채점 기준은 DAY7에 확정한 것을 그대로 사용합니다. V2 평가에서는 케이스를 20개로 확장했습니다(Basic 2개·Edge 1개·Safety 1개 추가).

- **채점 지표**: Pass Rate = 맞힌 케이스 수 ÷ 전체 케이스 수 × 100(%)
- **통과 조건**: ① 라벨 일치(사기/정상) AND ② 기대 신호 부분집합 포함
- **임계값**: 45점 이상 → 사기 판정, 44점 이하 → 정상 판정
- **평가셋**: `tests/cases.json`

## 근거

V2의 핵심 설계는 두 가지입니다.

**① 고위험 키워드 티어 (highRiskPatterns + highRiskBonus)**

단독 신호라도 실제 피해 사례에서 빈도가 높은 특정 키워드가 포함되면 추가 점수를 부여합니다. 티어별 보너스는 아래와 같습니다.

| 신호 유형 | 고위험 키워드 예시 | 기본 가중치 | 고위험 보너스 | 최종 점수 |
|-----------|-------------------|-------------|---------------|-----------|
| pressure | 영장, 체포, 구속, 계정정지, 압류, 벌금 | 16 | +30 | 46 |
| url | bit.ly, tinyurl, han.gl, me2.kr 등 단축URL | 18 | +30 | 48 |
| tooGood | 원금보장, 수익보장, 고수익, 확정수익 | 12 | +15 | 27 |

**② 콤보 보너스 (신호 조합 추가 점수)**

두 신호가 동시에 감지될 때 추가 점수를 부여합니다. V1에서 유지하던 콤보 2종에 신규 3종을 추가했습니다.

| 콤보 | 보너스 | 대표 패턴 |
|------|--------|-----------|
| url + money | +10 | 링크 + 금전 요구 |
| impersonation + personalInfo | +12 | 기관 사칭 + 개인정보 탈취 |
| money + pressure | +8 | 가족 사칭 송금 압박 |
| offPlatform + tooGood | +12 | 카톡 리딩방 투자 유도 |
| impersonation + pressure | +8 | 공문·영장 협박 |

**설계 원칙 — 오탐 방지**

임계값(45)과 기존 단일 가중치는 건드리지 않았습니다. 특정 케이스 텍스트를 하드코딩하거나 점수를 정밀 맞춤하지도 않았습니다. 고위험 티어와 콤보라는 **일반 규칙만 추가**해 새로운 사기 문자에도 그대로 적용될 수 있게 했습니다. 정상 문자의 오탐(거짓 경보)은 V1과 동일하게 0건을 유지했습니다.

결과적으로 V1 대비 +15%p를 달성하고, 목표 기준 85%를 넘어 90%를 기록했습니다.

## 남은 약점 — 정직한 한계

V2로 해결되지 않는 케이스가 2건 있습니다. 두 케이스 모두 규칙 기반 엔진이 구조적으로 풀 수 없는 **맥락 이해 문제**입니다.

| ID | 유형 | 실패 종류 | 원인 |
|----|------|-----------|------|
| case-h01 | Edge | 미탐(정탐 실패) | 지인 사칭 사회공학: "파일 링크 눌러봐" — 명시적 URL 패턴도, 사기 키워드도 없음. 0점 → 정상 오판. |
| case-h02 | Safety | 오탐(거짓 경보) | 드라마 감상 대화: "경찰·영장·체포" — 고위험 키워드가 있어 74점 → 사기 오판. |

두 케이스 모두 **글자가 아니라 대화의 맥락을 이해해야** 잡을 수 있습니다. 규칙 기반 엔진은 문맥을 이해하지 못하므로, DAY9 RAG식 키워드 확장으로도 이 한계는 해소되지 않습니다. 향후 LLM을 보강했을 때 풀 수 있는 과제로 남기고, DAY15 실사용 재검증에서 재확인합니다.
