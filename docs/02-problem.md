# 문제 정의 (problem.md)

> DAY 2 · AI 사기 메시지 방패 팀 최종 확정 · 작성자: 이도윤 (PM)

## 확정된 문제 한 문장

> 사기 문자·중고거래 채팅 메시지를 받은 일반 사용자가, 사기인지 정상인지 스스로 판단하지 못해 피해를 입거나 불필요한 불안을 겪는다.

## 대상 사용자

- 스마트폰 문자(SMS)나 중고거래 채팅(당근마켓·번개장터 등)을 이용하는 10~40대 일반 사용자.
- 특히 디지털 경험이 적어 '이게 사기인가?' 판단이 어려운 첫 중고거래 이용자, 어르신.
- 최유나 어머니처럼 "무서워서 아예 문자를 안 여는" 과잉 불안 상태에 처한 사람도 포함합니다.

## 핵심 페인 포인트

- 판단이 서지 않아 10분 넘게 고민하다 정상 안내 문자까지 무시하는 상황이 생깁니다.
- 카카오톡 단체방에 스크린샷을 올려 답변을 기다리지만, 빠른 답변이 보장되지 않고 판단 결과도 사람마다 다릅니다.
- 포털 검색으로 문자 내용을 확인해도, 최신 사기 수법은 검색 결과에 없는 경우가 많습니다.
- 스마트폰 기본 스팸 필터는 단순 키워드 차단 방식이라 새 수법에 금방 뚫립니다.
- "의심은 하는데 판단을 못 하는" 중간 상태가 가장 위험하며, 이를 빠르게 해소해줄 도구가 없습니다.

## AI 적합성 판단 (왜 AI로 풀기 적합한가)

AI 적합성 필터 4가지를 모두 통과합니다.

1. **사람마다 판단이 갈리는 언어 작업인가?** 예. "이 문장 사기야?"는 사람마다 다르게 판단하며, 새 수법일수록 더 갈립니다. 언어의 패턴·맥락을 읽는 LLM이 일관된 기준을 제공할 수 있습니다.
2. **반복적으로 일어나는가?** 예. 중고거래 커뮤니티에 "이 문자 사기인가요?" 글이 주 3~4건 꾸준히 올라옵니다. 매일 수많은 사람이 같은 고민을 반복합니다.
3. **정답이 명확해 채점(Pass Rate)이 가능한가?** 예. "사기 / 정상" 두 라벨로 결과를 판정할 수 있어, 모델이 맞혔는지 틀렸는지 숫자로 측정할 수 있습니다. 평가·개선 루프를 16일 안에 돌릴 수 있습니다.
4. **16일 안에 끝낼 범위인가?** 예. 텍스트 입력 → 사기/정상 분류 + 위험신호 추출 + 경고 문장 출력으로 범위를 명확히 제한하면 완주 가능합니다.

## 후보 A·B·C 중 이 문제를 고른 이유

세 후보를 AI 적합성 필터로 비교한 결과, 후보 A가 유일하게 4가지 필터를 모두 통과했습니다.

- **후보 B(학교 공지 쉬운말 변환)**: 사회적 가치는 크지만, "쉬운 말로 잘 변환됐는가?"의 정답이 여러 개입니다. 어떤 번역이 더 좋은지 사람마다 다르게 평가하기 때문에, 16일 안에 Pass Rate로 개선을 측정하기 어렵습니다.
- **후보 C(리뷰 → 개선 리포트)**: 리뷰 요약의 품질을 채점하는 기준이 모호합니다. "이 요약이 정확한가?"를 판단할 명확한 라벨이 없어, 평가 설계 자체에 시간이 많이 걸립니다.
- **후보 A(사기 탐지)**: '사기/정상' 두 라벨로 채점 기준이 명확합니다. 테스트 케이스를 만들고 Pass Rate를 측정해 개선 루프를 돌리기 가장 쉬운 구조입니다.

최유나(평가 담당)가 "B·C는 좋은 답이 여러 개라 16일 안에 평가·개선 루프를 완성하기 어렵다"는 근거를 제시했고, 팀 전체가 A로 수렴했습니다.

## 범위 (이번 16일에 다루는 것 / 다루지 않는 것)

- **다룬다:**
  - 문자(SMS)·중고거래 채팅 텍스트를 '사기 / 정상'으로 분류하기
  - 사기 가능성이 높은 위험신호(예: 외부 링크 유도, 개인정보 요청 문구)를 텍스트에서 추출하기
  - 사용자가 이해하기 쉬운 경고 메시지 출력하기 (예: "이 메시지는 외부 링크 클릭을 유도하고 있어 주의가 필요합니다.")
  - 평가 케이스 최소 16개(사기 / 정상 혼합) 설계 및 Pass Rate 측정
- **다루지 않는다:**
  - 이미지(스크린샷) 판독 — 텍스트만 다룹니다.
  - 통화 음성 분석
  - 실시간 자동 차단 기능 (사용자가 메시지를 붙여넣어 확인하는 방식)
  - 신고 자동 접수 또는 수사기관 연동
  - 다국어(한국어 외) 지원
