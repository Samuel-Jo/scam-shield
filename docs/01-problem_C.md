> 발굴: 이도윤 · DAY1

# 문제 후보 C — 리뷰 → 사장님 개선 리포트

## 누구의 문제인가 (대상 사용자)

- 배달 앱(배달의민족·쿠팡이츠 등) 또는 네이버 지도에 가게가 등록된 소상공인 자영업자.
- 주로 1인 운영이거나 직원 1~2명 규모의 작은 가게 사장님.
- 리뷰를 읽고는 싶지만 시간이 없고, 어떤 점을 고쳐야 할지 막막한 분들.

## 어떤 상황에서 겪는가

- 영업이 끝난 늦은 밤, 누적된 리뷰 수십 개를 보려고 앱을 열었다가 피곤해서 그냥 닫는 상황.
- 별점 3점짜리 리뷰가 달렸는데 "그냥 별로였어요" 한 줄뿐이라 무엇을 고쳐야 할지 알 수 없는 상황.
- 리뷰를 읽었지만 "맛있어요", "별로예요"처럼 감상 위주여서, '포장이 문제인지 맛이 문제인지 양이 문제인지' 구분이 안 되는 상황.

## 반복적으로 겪는 페인 포인트 (관찰/인터뷰 근거)

- 이도윤이 동네 분식집 사장님(운영 3년차)을 인터뷰했을 때, "리뷰는 쌓이는데 뭘 고쳐야 할지 모르겠다"는 말이 나왔습니다. 리뷰 탭을 마지막으로 열어본 게 2주 전이었고, "볼 시간이 없다"고 했습니다.
- 카페 사장님 B는 리뷰 200개가 쌓였지만 직접 읽기 어려워서 조카에게 부탁해 요약을 받았습니다. 조카의 주관적 정리라 신뢰가 낮았고, 다음번에는 또 부탁하기 미안했다고 했습니다.
- 자영업자 커뮤니티 게시글을 보면 "나쁜 리뷰를 보면 기분이 상해서 아예 안 본다"는 글이 반복됩니다. 감정적으로 읽기 어렵다는 것이 리뷰 확인의 심리적 장벽이었습니다.
- 배달 앱의 기본 통계 기능은 별점 평균만 보여주고, 어떤 메뉴·어떤 키워드에서 불만이 나오는지는 알려주지 않습니다. 사장님 입장에서 "뭘 고쳐야 하나"는 스스로 해석해야 합니다.

## 지금은 어떻게 (불편하게) 해결하고 있나

- 리뷰를 아예 보지 않습니다. 개선 기회를 놓칩니다.
- 시간이 날 때 직접 읽으며 손으로 메모합니다. 매일 지속하기 어렵고 주관이 들어갑니다.
- 가족·지인에게 리뷰를 읽어달라고 부탁합니다. 비정기적이고 일관성이 없습니다.
- 별점이 떨어지면 원인을 모른 채 메뉴를 바꾸거나 가격을 내리는 시도를 합니다. 원인을 특정하지 못하므로 효과가 불확실합니다.

## 이 문제가 중요한 이유

- 국내 소상공인 수는 수백만 명이며, 배달 앱 리뷰는 매출에 직결되는 데이터임에도 대부분 활용되지 않고 있습니다.
- "읽기 → 패턴 파악 → 개선 항목 도출"이라는 세 단계 모두 반복 언어 작업입니다. NLP로 자동화할 여지가 큽니다.
- 리뷰를 분석해서 "이번 주 포장 불만 3건, 대기 시간 불만 2건" 형태로 정리해주면, 사장님이 구체적인 행동을 취할 수 있습니다.
- 고객 의견을 반영해 개선하는 선순환이 만들어지면, 별점 상승 → 매출 증가로 이어지는 실질적 효과를 기대할 수 있습니다.
