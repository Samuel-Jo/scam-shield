# Docs as Code — 폴더 및 파일 명명 규칙

> 이 문서는 어떤 DAY에 어떤 파일이 `docs/`에 생기는지 미리 안내합니다.
> 아직 존재하지 않는 파일은 "앞으로 생길 파일"입니다. DAY0에는 만들지 않습니다.

---

## 기본 원칙

1. **모든 산출물 문서는 `docs/` 폴더 안에 .md 파일로 저장합니다.**
2. **파일명 앞에 두 자리 번호 접두어를 붙입니다.** 번호 순서가 곧 작성 순서입니다.
3. **파일명은 영문 소문자와 밑줄(`_`)만 사용합니다.** 공백이나 한글은 쓰지 않습니다.
4. **한 파일에는 하나의 주제만 담습니다.** 파일이 길어지면 분리합니다.

## 교육자료(가이드)는 `docs/guides/`에 모읍니다

산출물 문서(번호 접두어)와 별개로, **각 과정을 비전공 학생용으로 풀어 쓴 교육자료**는
`docs/guides/` 폴더에 주제별 이름으로 둡니다. (한 과정이 끝날 때마다 1개씩 추가)

| 파일 | 다루는 과정 |
|------|-------------|
| `guides/github-collaboration.md` | GitHub 협업(저장소 시작·공유·PR·통합) |
| `guides/problem-discovery.md` | 문제 후보 도출·페인 포인트 관찰법 (DAY1) |

## 자주 묻는 질문

**Q. 번호가 같은 파일이 여러 개여도 되나요?**
같은 날(DAY)에 여러 파일이 생길 수 있습니다. 번호가 같아도 파일명이 다르면 문제없습니다.

**Q. 문서를 갱신할 때 파일을 새로 만드나요?**
같은 파일을 수정합니다. 버전 변경 내역은 파일 안 상단에 버전 표기로 남깁니다 (예: `v1 → v2`). 중요한 변경은 `13-changelog.md`에도 기록합니다.

**Q. `src/`와 `tests/`는 언제 생기나요?**
`src/`는 DAY5(최초 코드 구현)부터, `tests/`는 DAY7(평가 케이스 설계)부터 파일이 생깁니다. DAY0 현재는 폴더만 예약된 상태입니다.
