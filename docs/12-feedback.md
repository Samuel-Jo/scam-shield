---

## 데모 개요

- **일시**: DAY 12 데일리싱크 직후
- **시연자**: 정하린 (UX lead)
- **대상**: DAY 4 초기 테스터 김지우 외 2명 (총 3명, 비전공 대학생)
- **시연 버전**: V2 CLI (규칙 기반 + 고위험 티어 + 콤보 보너스 + 가드레일 4종)
- **시연 방식**: 실제 사기 문자 예시 3종과 정상 문자 1종을 직접 입력해 결과를 함께 확인

---

## 체크리스트 점검 결과

- [x] 페인 포인트가 실제 시연에서 해결되었는가? — **부분 해결**. 위험도 점수와 신호 하이라이트는 의도대로 작동했으나, "그래서 어떻게 해야 하나"라는 행동 안내가 부족하다는 반응이 있었습니다.
- [x] Self-correction loop가 원활히 작동하는가? — **확인됨**. 가드레일 4종(guardInput · detectPromptInjection · guardOutput · maskPII)이 모두 작동하고, 출력 검증이 단정 표현을 자동 완화했습니다.
- [x] PRD/TRD 갱신이 필요한 항목이 정의되었는가? — **예**. 행동 안내 문구(PRD v2), 오탐 한계 안내(TRD/한계 안내 문서) 두 항목이 DAY 13 갱신 대상으로 확정되었습니다.

## 다음 Iterate에서 반영할 항목

1. **행동 안내 한 줄 추가** — 위험도 경고 바로 아래에 "링크를 누르지 마세요", "금융감독원(1332) 또는 경찰청(112)에 신고하세요" 같은 다음 행동을 명시합니다. PRD v2에 필수 기능으로 추가하고, DAY 13 갱신 시 반영합니다.

2. **오탐 한계 안내 문구 추가** — "드라마 대화, 소설 텍스트처럼 사기 키워드가 포함된 일상 문자는 오탐이 발생할 수 있습니다"라는 한계를 결과 화면 하단 또는 안내 영역에 짧고 담백하게 표시합니다. TRD v2와 한계 안내 문서에 반영하고, DAY 13 갱신 시 포함합니다.

3. **README / PRD v2 갱신 — DAY 13에** — 위 두 항목을 포함해 PRD와 TRD를 v2로 갱신합니다. Changelog에도 피드백 근거를 명시합니다.
