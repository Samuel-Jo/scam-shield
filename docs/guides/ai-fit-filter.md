# AI 적합성 필터로 문제 고르기 (데모 팀 DAY2 실전)

> DAY 2 교육자료 · 대상: 비전공 학생 · 작성: AI 사기 메시지 방패 팀

---

## 왜 아무 문제나 AI로 풀면 안 되는가

AI는 만능 도구가 아닙니다. 마치 계산기가 "어느 학교에 지원할까?"를 대신 결정해 줄 수 없는 것처럼, 어떤 문제는 AI가 잘 풀고, 어떤 문제는 AI보다 사람이 더 잘 풉니다.

비전공 학생이 16일 안에 완성할 수 있는 AI 프로젝트에는 두 가지 조건이 필요합니다.

1. **AI가 실제로 도움이 되는 문제여야 합니다.** AI가 이미 잘하는 언어 패턴 인식, 분류, 요약 같은 작업에 맞아야 합니다.
2. **16일 안에 "좋아졌다"를 숫자로 증명할 수 있어야 합니다.** 막연히 "더 나은 것 같다"로 끝나면 프로젝트가 완성됐는지 알 수 없습니다.

이 두 조건을 빠르게 확인하는 방법이 바로 **AI 적합성 필터**입니다.

---

## 4가지 필터 질문

아래 4가지 질문에 "예"가 많을수록 AI 프로젝트로 만들기 좋은 문제입니다.

| # | 질문 | 왜 중요한가 |
|---|------|------------|
| 1 | **사람마다 판단이 갈리는 언어 작업인가?** | AI(LLM)는 글을 읽고 패턴을 찾는 데 강합니다. 사람마다 답이 다른 언어 판단 문제라면 AI가 일관된 기준을 제공할 수 있습니다. |
| 2 | **반복적으로 일어나는가?** | 한 번만 겪는 문제는 AI보다 사람이 직접 해결하는 게 빠릅니다. 매일·매주 같은 고통이 반복되어야 자동화의 가치가 생깁니다. |
| 3 | **정답이 명확해 채점(Pass Rate)이 가능한가?** | "맞았다/틀렸다"를 숫자로 셀 수 있어야 개선했을 때 얼마나 나아졌는지 증명할 수 있습니다. 정답이 여러 개이거나 주관적이면 채점 기준을 만드는 데만 시간이 다 갑니다. |
| 4 | **16일 안에 끝낼 범위인가?** | 범위가 너무 크면 완주를 못 합니다. 입력과 출력을 좁게 정의할수록 완성 확률이 올라갑니다. |

## 우리 팀이 A·B·C를 필터로 비교한 실제 표

| 필터 질문 | 후보 A: 사기 탐지 | 후보 B: 공지 변환 | 후보 C: 리뷰 리포트 |
|-----------|------------------|------------------|---------------------|
| 사람마다 판단이 갈리는 언어 작업인가? | ○ 사기/정상 판단은 사람마다 다름 | ○ 쉬운 말 변환은 언어 작업 | ○ 리뷰 요약도 언어 작업 |
| 반복적으로 일어나는가? | ○ 매주 수백만 건 발생 | ○ 매 학기 공지 반복 | △ 리뷰는 쌓이지만 사장님이 확인 빈도가 낮음 |
| 정답이 명확해 채점이 가능한가? | **○ '사기/정상' 두 라벨로 Pass Rate 측정 가능** | **× '좋은 변환'이 여러 개라 채점 기준 만들기 어려움** | **× 요약 품질 판단이 주관적이라 채점 기준 설계 부담 큼** |
| 16일 안에 끝낼 범위인가? | ○ 텍스트 분류 + 위험신호 추출로 범위 제한 가능 | △ 공지 종류가 다양해 범위 설정이 까다로움 | △ 리뷰 분석 항목이 늘어날수록 범위가 커짐 |
| **종합** | **4/4 통과** | **2/4 (채점 불가·범위 주의)** | **2/4 (채점 불가·반복성 주의)** |

**범례**: ○ 통과 / △ 조건부 / × 통과 못함

---

## A 선택 결론

4가지 필터를 모두 통과한 후보는 **A(사기 탐지)** 뿐입니다.

B(공지 변환)와 C(리뷰 리포트)는 사회적으로 의미 있는 문제이지만, "채점 가능성" 필터에서 막혔습니다. 16일 안에 평가·개선 루프를 완성하려면 채점이 가능해야 하는데, 두 후보는 정답이 여러 개라 채점 기준을 만드는 것 자체가 별도 과제가 됩니다.

팀이 최종 확정한 문제 정의서는 여기서 확인하세요: [docs/02-problem.md](../02-problem.md)

## 채점 가능성이 왜 중요한가

비전공 학생 팀이 16일 안에 반드시 해야 하는 일이 있습니다. "AI가 좋아졌다"를 증명하는 것입니다.

증명하려면 측정해야 하고, 측정하려면 채점 기준이 있어야 합니다.

> 비유: 시험 점수를 올렸다고 주장하려면, 먼저 시험지와 정답지가 있어야 합니다.
> AI 프로젝트도 같습니다. 정답 라벨이 없으면 "점수가 올랐는지"를 알 방법이 없습니다.

사기 탐지는 "사기 = 1, 정상 = 0" 두 가지 라벨로 정답지를 만들 수 있습니다. 예를 들어 테스트 케이스 20개를 준비하고, 모델이 몇 개를 맞히는지 세면 됩니다. 이것이 **Pass Rate**입니다.

- DAY 6: 첫 모델 → Pass Rate 65%
- DAY 9: 프롬프트 개선 → Pass Rate 80%
- DAY 13: 가드레일 추가 → Pass Rate 90%

이처럼 숫자가 바뀌는 것을 보여줄 수 있어야 "AI가 실제로 개선됐다"고 말할 수 있습니다.

## 따라하기: 내 팀의 후보를 필터로 점검하는 방법

1. 후보 문제를 적습니다. (후보 A, B, C)
2. 위 표의 4가지 질문을 하나씩 적용합니다.
3. ○ / △ / × 로 표시합니다.
4. ○가 3개 이상인 후보를 고릅니다.
5. **특히 "채점 가능성" 필터를 통과하지 못하면**, 평가 설계를 먼저 해결하지 않는 한 후순위로 미루세요.
6. 선택한 이유를 `02-problem.md`의 "후보 A·B·C 중 이 문제를 고른 이유" 섹션에 기록합니다.
