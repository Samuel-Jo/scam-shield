# 좋은 문제 찾기 — 페인 포인트 관찰법 (데모 팀 DAY1 실전)

> 이 문서는 우리 팀이 DAY1에 실제로 한 일을 바탕으로 쓴 교육자료입니다.
> 코드는 한 줄도 없습니다. 문제를 잘 찾는 것이 먼저입니다.

---

## 1. 왜 코딩보다 '문제'가 먼저인가

프로젝트를 처음 시작할 때 많은 사람이 이렇게 생각합니다.

> "일단 뭘 만들지 정하고, 바로 코딩하자."

하지만 이렇게 하면 완성한 뒤에야 "이걸 쓸 사람이 없네"를 깨닫게 됩니다.
마치 식당을 열고 나서 "이 동네 사람들이 이 음식을 좋아하나?"를 물어보는 것과 같습니다.

좋은 프로젝트는 반대 순서로 시작합니다.

```
문제 발견 → 문제 이해 → 해결책 설계 → 구현
```

코딩은 마지막입니다. 문제가 선명할수록 코드가 짧아지고, 쓸모 있는 결과물이 나옵니다.

---

## 2. '만들고 싶은 것' vs '누군가 불편한 것'

이 둘은 비슷해 보이지만 완전히 다릅니다.

| 만들고 싶은 것 | 누군가 불편한 것 |
|----------------|-----------------|
| "AI 챗봇을 만들고 싶다" | "공지를 읽어도 무슨 말인지 몰라서 마감을 놓쳤다" |
| "리뷰 분석 앱이 있으면 멋있을 것 같다" | "리뷰 200개가 쌓였는데 볼 시간이 없다" |
| "사기 탐지 서비스를 만들어보고 싶다" | "이 문자 눌러도 되는지 10분째 고민 중이다" |

왼쪽은 '내 관심'에서 출발합니다. 오른쪽은 '남의 고통'에서 출발합니다.
오른쪽에서 출발한 프로젝트가 실제로 쓰입니다.

핵심 질문은 딱 하나입니다.

> "지금 이 순간 어떤 사람이 이 문제 때문에 불편해하고 있는가?"

## 3. 페인 포인트 관찰·인터뷰 팁 (비전공자도 할 수 있는 방법)

### 관찰: 온라인에서 찾기

사람들은 불편할 때 인터넷에 씁니다. 그 글이 데이터입니다.

- **커뮤니티 게시판**: 에브리타임·네이버카페·레딧에서 "이거 어떻게 하나요?", "불편한데 방법 없나요?" 같은 글을 찾습니다.
- **앱 리뷰**: Play스토어·앱스토어에서 별점 2~3점짜리 리뷰를 읽습니다. 불만이 가장 솔직하게 나옵니다.
- **단체방/오픈채팅**: 같은 질문이 반복되면 그 질문이 페인 포인트입니다.

박서준은 중고거래 커뮤니티에서 "이 문자 사기인가요?" 글이 주 3~4건 올라오는 것을 2주 동안 관찰했습니다.

### 인터뷰: 주변 사람에게 물어보기

인터뷰라고 거창하게 생각할 필요 없습니다. 카카오톡 메시지 3~4개면 충분합니다.

좋은 질문의 패턴은 이렇습니다.

- "최근에 ○○하다가 불편했던 적 있어요?" (경험을 묻기)
- "그때 어떻게 해결했어요?" (현재 해결 방법 파악)
- "그 방법이 번거롭거나 아쉬운 점은?" (진짜 페인 포인트 파악)

하지 않아야 할 질문은 이것입니다.

- "이런 앱이 있으면 쓸 것 같아요?" → "네"라는 답이 자동으로 나오는 질문입니다. 행동 의향은 실제 행동과 다릅니다.

이도윤은 동네 분식집 사장님에게 5분 인터뷰를 해서 "리뷰를 마지막으로 열어본 게 2주 전"이라는 구체적인 정보를 얻었습니다. 수치가 있으면 문서가 살아납니다.

### 기록: 장면으로 적기

페인 포인트를 적을 때는 "불편하다"는 감정이 아니라 "어떤 상황에서 어떤 행동을 했더니 어떤 결과가 생겼다"는 장면으로 씁니다.

| 추상적 기록 (나쁜 예) | 구체적 기록 (좋은 예) |
|----------------------|-----------------------|
| 사기 문자 때문에 불편하다 | 택배 문자를 받고 눌러야 할지 10분 고민하다 그냥 무시했더니, 실제로는 진짜 택배 안내였다 |
| 공지가 어렵다 | 수강신청 전날 공지를 읽었는데 무슨 뜻인지 몰라 선배 카톡을 밤새 기다렸다 |

장면으로 적으면 나중에 해결책을 설계할 때 "어디서 도와줄까"가 명확해집니다.

## 4. 후보를 3개 뽑는 이유

처음부터 하나를 고르면 "이게 최선인가?" 의심이 남습니다.
너무 많으면 비교가 오히려 어렵습니다.

3개는 비교하기에 딱 맞는 숫자입니다.

- 각 후보를 나란히 놓고 보면 공통점과 차이점이 보입니다.
- "어떤 기준으로 고를까"를 팀이 함께 논의할 수 있습니다.
- DAY2의 'AI 적합성 필터'를 3개에 각각 적용하면 기준이 일관성 있게 작동합니다.

우리 팀은 DAY0 데일리싱크에서 각자 관찰해온 후보를 1인 1개씩 가져와 3개를 확보했습니다.

## 5. 우리 팀 실제 사례 — 후보 A / B / C

### 후보 A — 사기 메시지 탐지 (발굴: 박서준)

중고거래·문자 메시지에서 오는 피싱·스미싱을 빠르게 판별하지 못해, 클릭해야 할지 말아야 할지 결정 자체가 스트레스가 되는 문제.

상세 내용: [docs/01-problem_A.md](../01-problem_A.md)

### 후보 B — 학교 공지 쉬운말 변환 (발굴: 정하린)

신입생·유학생이 대학 행정 공지의 문어체를 이해하지 못해, 중요한 마감을 놓치거나 매번 선배·조교에게 같은 질문을 반복하는 문제.

상세 내용: [docs/01-problem_B.md](../01-problem_B.md)

### 후보 C — 리뷰 → 사장님 개선 리포트 (발굴: 이도윤)

소상공인이 쌓인 고객 리뷰를 읽을 시간이 없어서, 어떤 점을 개선해야 할지 파악하지 못하고 있는 문제.

상세 내용: [docs/01-problem_C.md](../01-problem_C.md)

## 6. 다음 DAY2 예고 — AI 적합성 필터로 1개 선택

DAY1은 후보 3개를 "발굴·기록"하는 단계입니다. 아직 어떤 것이 정답인지 정하지 않습니다.

DAY2에서는 각 후보에 **AI 적합성 필터**를 적용해서 최종 1개를 선택합니다.
AI 적합성 필터란, "이 문제가 AI(특히 NLP)로 풀기에 적합한가?"를 기준으로 후보를 거르는 체크리스트입니다. 예를 들어 이런 질문들입니다.

- 반복적인 언어 처리 작업인가?
- 사람마다 판단이 다를 수 있는 영역인가?
- 16일 안에 의미 있는 결과를 낼 수 있는 범위인가?

지금은 "이런 기준으로 고를 것"이라는 것만 알아두면 됩니다. 선택은 DAY2에 합니다.
