## 문서 버전

- v1 (DAY 4) → **v2 (DAY 13)** — DAY 12 피드백 반영 갱신

> **v2 변경 요약**: ①행동 안내 한 줄(필수 기능으로 추가), ②오탐 한계 안내 문구(Out of Scope 아래 별도 한계 안내 항목 신설). 나머지 기존 내용은 유지.

---

## 한 줄 제품 설명

> 의심스러운 문자나 채팅 내용을 붙여넣으면, 위험도 점수와 위험 신호, 쉬운 경고를 3초 안에 보여주는 웹 도구.

---

## 핵심 기능 (우선순위 순)

1. [필수] 텍스트 입력 → 위험도 점수 산출: 사용자가 의심 문자·채팅 내용을 붙여넣으면 0~100점의 위험도 점수를 계산해 보여준다.
2. [필수] 위험 신호 하이라이트 + 이유 표시: 위험도를 높이는 구체적인 문구(예: "아래 링크에서 수정")를 텍스트 안에서 강조 표시하고, 왜 위험한지 한 줄로 이유를 함께 보여준다.
3. [필수] 쉬운 한 줄 경고: 전문용어 없이 누구나 이해할 수 있는 경고 문장을 결과 맨 위에 출력한다. (예: "외부 링크 클릭을 유도하는 전형적인 사기 문자 패턴입니다. 링크를 누르지 마세요.")
4. **[필수 · v2 신규] 행동 안내 한 줄**: 위험도 경고 바로 아래에 다음에 취해야 할 행동을 한 줄로 안내한다. 예: "링크를 누르지 마세요. 의심되면 금융감독원(1332) 또는 경찰청(112)에 신고하세요." — DAY 12 피드백 #1 반영. 박서준 구현 예정(`actionAdvice` 필드).
5. [선택] 예시 문구 체험: 사용자가 직접 문자를 갖고 있지 않아도, 미리 준비된 예시 문구(사기/정상 각 2~3개)를 버튼 한 번으로 입력창에 불러와 서비스를 체험할 수 있다. — DAY 12 피드백 #4, 이번 16일에는 차기 백로그로 유보. → 후속 정식 배포에서 반영됨(예시 버튼 10개, scam-shield-9uc.pages.dev).
6. [선택] Eval(Pass Rate) 표시: 현재 서비스가 라벨된 평가셋에서 몇 %를 맞히는지(Pass Rate)를 화면 하단에 표시해, 서비스의 신뢰 수준을 사용자가 직접 확인할 수 있다.

---

## 각 기능별 사용자 스토리

- 김지우(대학생)는 모르는 번호로 온 문자가 사기인지 빠르게 판단하기 위해, 문자 내용을 복사해 입력창에 붙여넣고 [확인] 버튼을 누를 수 있다.
- 김지우는 "왜 위험한지"를 이해하기 위해, 위험 신호가 강조된 텍스트와 한 줄 이유 설명을 결과 화면에서 확인할 수 있다.
- 김지우는 위험 판정을 받은 뒤 "그래서 어떻게 해야 하나"를 알기 위해, 결과 화면에서 구체적인 행동 안내(신고 전화번호 포함)를 확인할 수 있다. **(v2 신규)**
- 처음 방문한 사용자는 서비스가 어떻게 작동하는지 바로 체험하기 위해, 예시 사기 문구 버튼을 눌러 샘플 분석 결과를 볼 수 있다. (선택 기능 — 차기 백로그)
- 최유나(평가 담당)는 서비스의 신뢰 수준을 확인하기 위해, 화면에서 Pass Rate 수치를 직접 조회할 수 있다. (선택 기능 — 시간 남으면 구현)

## 화면/흐름 개요

사용자가 거치는 화면 흐름은 03-scenario.md의 이상적인 흐름과 일치합니다.

```
[1단계] 입력 화면
  - 상단: 서비스 이름 + 한 줄 설명 ("의심 문자를 붙여넣으세요")
  - 중앙: 텍스트 입력창 (여러 줄 입력 가능)
  - 하단: [확인] 버튼

        ↓ (버튼 클릭)

[2단계] 분석 중
  - 로딩 상태 표시 (예: "분석 중..." 문구 또는 스피너)
  - 목표: 3초 이내 완료

        ↓ (분석 완료)

[3단계] 결과 화면
  - 상단: 쉬운 한 줄 경고 (예: "이 문자는 사기 가능성이 높습니다. 링크를 누르지 마세요.")
  - 중앙 상-1: 위험도 점수 게이지 (0~100, 숫자 + 색상 막대)
  - 중앙 상-2: [v2 신규] 행동 안내 한 줄
      예) "링크를 누르지 마세요. 의심되면 금융감독원(1332) 또는 경찰청(112)에 신고하세요."
  - 중앙 하: 위험 신호 하이라이트 + 이유 목록
      예) "아래 링크에서 수정" → 외부 링크 클릭 유도 패턴
          "개인정보를 입력" → 개인정보 수집 시도 패턴
  - 하단: [다시 확인하기] 버튼 (입력 초기화)
  - 하단(선택): Pass Rate 표시
```

## 이번 버전에서 하지 않는 것 (Out of Scope)

- 이미지(스크린샷) 판독: 텍스트를 직접 붙여넣는 방식만 지원합니다. 화면 캡처 이미지를 업로드해 내용을 읽어내는 기능은 범위 밖입니다.
- 통화·음성 분석: 전화 사기(보이스피싱) 음성 파일이나 통화 내용 분석은 포함하지 않습니다.
- 실시간 자동 차단: 문자를 받는 즉시 자동으로 차단하거나 알림을 보내는 기능은 이번 버전에서 만들지 않습니다. 사용자가 직접 붙여넣어 확인하는 방식으로만 작동합니다.
- 신고 자동 접수: 사기 문자를 수사기관·통신사에 자동으로 신고하는 연동 기능은 범위 밖입니다.
- 로그인 필수화: 회원 가입·로그인 없이 누구나 바로 사용할 수 있어야 합니다. 인증 시스템은 V1에서 만들지 않습니다.
- 결과 가독성 색상 강조(등급 강조 UI): DAY 12 피드백 #3 — 빨강/초록 색 구분. 16일 차기 백로그로 유보했으며, 후속 정식 배포에서 반영됨.
- 예시 문구 체험 버튼: DAY 12 피드백 #4 — 차기 백로그로 유보했으며, 후속 정식 배포에서 반영됨(예시 버튼 10개).

## 한계 안내 (v2 신규 · DAY 12 피드백 #2 반영)

이 서비스는 규칙 기반 엔진으로 작동합니다. 다음과 같은 경우 오탐(정상 문자를 위험으로 잘못 판단)이 발생할 수 있습니다.

- 드라마·소설 내용을 복사한 텍스트 (예: "영장", "체포", "입금" 등의 단어가 포함된 일상 대화)
- 사기 키워드와 유사한 표현이 포함된 정상적인 업무·공지 문자

결과 화면 하단에 "이 서비스는 참고용이며, 일부 정상 문자가 위험으로 잘못 분류될 수 있습니다. 최종 판단은 사용자 본인이 합니다."라는 안내 문구를 항상 표시합니다.

## 성공 지표 (success.md 연동)

03-success.md에서 확정한 성공 기준과 직접 연동됩니다.

- **Pass Rate ≥ 85%**: 라벨된 평가셋(사기·정상 혼합, 20개 케이스)에서 서비스가 정확히 분류한 비율이 85% 이상이어야 합니다. DAY 13 현재 실측 **90%(18/20)** 달성.
- **판단 소요 시간 ≤ 5초**: 사용자가 문자를 붙여넣고 결과를 보기까지 5초 이내. 서비스 응답 시간 + 사용자가 결과를 읽고 판단을 내리는 시간을 포함합니다.
- **판단 시점**: DAY 15 최종 Eval에서 두 조건을 동시에 충족해야 성공으로 기록합니다.
