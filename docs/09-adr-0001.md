# ADR-0001: 규칙 기반 V2(고위험 티어+RAG식 키워드 KB) 채택, LLM/Fine-tuning 보류

> DAY 9 · AI 사기 메시지 방패 팀 · 담당: 박서준(개발) · 최유나(평가/데이터)

- 채택

## 맥락 (어떤 상황에서 결정이 필요했나)

- DAY7에 V1 규칙 엔진의 Pass Rate를 처음 실측한 결과 **75.0%(12/16)**로, 팀이 DAY3에 합의한 목표 85%에 10%p 미달했습니다.
- V1 실패 4건을 분석하니 공통 원인이 명확했습니다. "영장·체포"처럼 강력한 위험 신호 단독으로는 임계값 45점을 넘지 못하거나, "원금보장·카카오톡 리딩방" 조합도 합산 점수가 부족한 구조적 한계였습니다.
- DAY3에 목표를 이원화했습니다. 1차: V1 기준 85%, 2차 도전: DAY8 이후 V2에서 90%.
- 16일 일정과 예산 제약이 있습니다. 외부 LLM API 호출은 월정 비용이 발생하고, Fine-tuning은 충분한 라벨 데이터와 학습 시간이 필요합니다.

---

## 결정 (무엇을 선택했나)

- 규칙 엔진 안에 **신고 사례 기반 고위험 키워드 KB(highRiskPatterns 배열)**를 두고, 이를 탐지 시점에 참조·활용하는 방식으로 V2를 구현합니다.
- 이를 RAG(Retrieval-Augmented Generation)의 원리를 규칙 엔진에 적용한 것으로 봅니다. 외부 벡터DB 대신 로컬 배열로 관리하지만, "신고 사례를 찾아서 탐지에 참고시킨다"는 구조는 동일합니다.
- LLM 외부 호출과 Fine-tuning은 현재 16일 일정 범위 밖이므로 보류합니다.

---

## 이유 (왜 이 선택인가)

- **속도와 비용**: 고위험 키워드 KB는 배열 수정만으로 즉시 반영됩니다. 외부 API 호출 없이 로컬에서만 실행되어 비용이 0원입니다.
- **채점 가능성**: 규칙 기반이므로 변경 전후를 같은 평가셋으로 즉시 비교할 수 있습니다. "더 좋아진 것 같다"가 아니라 75% → 90%라는 숫자로 증명할 수 있습니다.
- **공격면**: 외부 LLM 호출이 없으므로 프롬프트 인젝션 공격 경로가 원천 차단됩니다. API 키 관리 부담도 없습니다.
- **확장성**: highRiskPatterns 배열에 새로운 키워드를 추가하는 것만으로 탐지 범위를 늘릴 수 있습니다. 실제 신고 사례가 접수될 때마다 배열에 추가하는 방식으로 운용할 수 있습니다.
- **결과**: V2로 실측 **90.0%(18/20)**를 달성했습니다. 목표 85%를 넘고, 2차 도전 목표 90%도 충족했습니다.

---

## 고려했지만 택하지 않은 대안

**① LLM 외부 호출 (GPT-4o, Claude 등)**

- 강점: 문장 맥락을 이해할 수 있어 case-h01(지인 사칭), case-h02(드라마 대화)처럼 키워드 없이 의도만으로 이루어진 경우도 판단 가능. 새로운 사기 수법에 대한 일반화 능력이 높음.
- 보류 이유:
  - 월정 API 비용이 발생합니다. 16일 프로젝트 예산 범위를 넘습니다.
  - 응답 지연이 발생해 "3초 안에 판단"이라는 성능 목표가 흔들릴 수 있습니다.
  - 프롬프트 인젝션 공격 경로가 생깁니다. 사기 문자 안에 AI 조종 명령이 숨어 있을 경우 탐지 결과가 조작될 위험이 있습니다.
  - API 키 관리가 필요합니다.
- 결론: 16일 범위 밖. 향후 LLM 보강 백로그로 등록.

**② Fine-tuning**

- 강점: 우리 도메인(한국 사기 문자)에 특화된 모델을 만들 수 있어 장기적으로 정확도 향상 가능.
- 보류 이유:
  - 충분한 라벨 데이터가 필요합니다. 현재 보유한 평가셋 20개는 학습 데이터로 사용하기에 턱없이 부족합니다.
  - 학습 실행과 결과 검증에 수 일이 필요합니다. 16일 일정 안에 완료하기 어렵습니다.
- 결론: 16일 범위 밖. 차기 개선 과제로 기록.

---

## 결과 / 영향

- V2 실측 Pass Rate: **90.0%(18/20)**. 목표 85% 달성, 2차 도전 목표 90% 달성.
- 태그별: Basic 8/8(100%), Edge 5/6(83.3%), Safety 5/6(83.3%).
- 정상 문자 오탐: case-h02(드라마 대화) 1건 외 0건. 기존 V1 수준 유지.
- **남은 한계 2건**:
  - case-h01: 지인 사칭 사회공학("파일 링크 눌러봐") — 키워드 없음, 0점, 미탐. 맥락 이해 필요.
  - case-h02: 드라마 대화("경찰·영장·체포") — 74점, 오탐. 문맥 구분 불가.
- 두 한계 케이스는 DAY15 실사용 재검증에서 재확인하고, LLM 보강을 차기 백로그로 등록합니다.
- highRiskPatterns 배열은 신고 사례 확인 시 항목을 추가하는 방식으로 지속 관리합니다.
