# src — 코드 폴더

DAY5에 V1 규칙 엔진이 추가됐습니다.

## 파일 구성

| 파일 | 설명 |
|---|---|
| `scamDetector.ts` | V1 규칙 기반 사기 탐지 엔진 (TypeScript, standalone) |
| `cli.mjs` | Node 즉시 실행용 CLI (순수 JS, 빌드 불필요) |

## 실행법

Node.js 설치 후 프로젝트 루트(demo-project)에서 실행합니다.

```bash
# 특정 문자 분석
node src/cli.mjs "분석할 메시지"

# 예) 스미싱 탐지
node src/cli.mjs "[국세청] 미납 세금 확인 http://bit.ly/x 지금 본인인증"

# 예) 정상 문자
node src/cli.mjs "내일 2시 강남역에서 직거래 가능할까요?"

# 인수 없이 실행하면 내장 예시 3개(스미싱/선입금/정상)를 자동 시연
node src/cli.mjs
```

## 탐지 규칙 요약 (V1 — 7종)

| 신호 타입 | 가중치 | 설명 |
|---|---|---|
| `money` | 22 | 선입금·계좌이체 등 금전 요구 |
| `personalInfo` | 22 | 인증번호·비밀번호·주민번호 요구 |
| `impersonation` | 20 | 국세청·경찰·은행 등 기관 사칭 |
| `url` | 18 | 단축URL·낯선 링크 포함 |
| `pressure` | 16 | "지금 당장·미납·긴급" 등 심리적 압박 |
| `offPlatform` | 12 | 안전결제 회피, 직거래·카톡 유도 |
| `tooGood` | 12 | 반값·원금보장·고수익 등 비현실적 조건 |

콤보 보너스:
- `url` + `money` 동시 탐지 → +10
- `impersonation` + `personalInfo` 동시 탐지 → +12

위험 등급 임계값:
- 70점 이상: 매우위험
- 45점 이상: 위험
- 20점 이상: 주의
- 20점 미만: 안전

## V1 한계 (규칙 기반의 한계)

1. **신조어·우회 표현에 약함**: "입금" 대신 "돈 좀 부쳐줘", "국*청" 같은 변형 표현은 탐지하지 못합니다.
2. **문맥 이해 없음**: 단순 키워드 매칭이라 정상 대화에서 우연히 포함된 단어도 신호로 잡을 수 있습니다.
3. **새로운 수법 미반영**: 규칙 사전에 없는 신종 사기 패턴은 탐지하지 못합니다.

DAY8~9에서 LLM/RAG를 활용한 V2 엔진으로 위 한계를 보강할 예정입니다.
