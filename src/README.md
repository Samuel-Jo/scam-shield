| 파일 | 설명 |

## 실행법

Node.js 설치 후 프로젝트 루트(demo-project)에서 실행합니다.

```bash
# 특정 문자 분석
node src/cli.mjs "분석할 메시지"

# 예) 스미싱 탐지
node src/cli.mjs "[국세청] 미납 세금 확인 http://bit.ly/x 지금 본인인증"

# 예) 정상 문자
node src/cli.mjs "내일 2시 강남역에서 직거래 가능할까요?"

# 인수 없이 실행하면 내장 예시 3개(스미싱/선입금/정상)를 자동 시연
node src/cli.mjs
```

## 탐지 규칙 요약 (V1 — 7종)

| 신호 타입 | 가중치 | 설명 |
|---|---|---|
| `money` | 22 | 선입금·계좌이체 등 금전 요구 |
| `personalInfo` | 22 | 인증번호·비밀번호·주민번호 요구 |
| `impersonation` | 20 | 국세청·경찰·은행 등 기관 사칭 |
| `url` | 18 | 단축URL·낯선 링크 포함 |
| `pressure` | 16 | "지금 당장·미납·긴급" 등 심리적 압박 |
| `offPlatform` | 12 | 안전결제 회피, 직거래·카톡 유도 |
| `tooGood` | 12 | 반값·원금보장·고수익 등 비현실적 조건 |

콤보 보너스:
- `url` + `money` 동시 탐지 → +10
- `impersonation` + `personalInfo` 동시 탐지 → +12

위험 등급 임계값:
- 70점 이상: 매우위험
- 45점 이상: 위험
- 20점 이상: 주의
- 20점 미만: 안전

## V1 한계 (규칙 기반의 한계)

1. **신조어·우회 표현에 약함**: "입금" 대신 "돈 좀 부쳐줘", "국*청" 같은 변형 표현은 탐지하지 못합니다.
2. **문맥 이해 없음**: 단순 키워드 매칭이라 정상 대화에서 우연히 포함된 단어도 신호로 잡을 수 있습니다.
3. **새로운 수법 미반영**: 규칙 사전에 없는 신종 사기 패턴은 탐지하지 못합니다.

DAY8~9에서 고위험 티어·RAG식 키워드 KB·콤보 보너스로 V2 엔진이 구현됐습니다. 웹앱 배포됨: scam-shield-9uc.pages.dev (`web/index.html`).

## DAY10 가드레일 (guardrails.mjs)

분류(detectScam) 로직 자체는 변경하지 않고, 입력 전처리·출력 후처리 계층으로만 안전장치를 추가합니다.

| 가드레일 | 함수 | 동작 요약 |
|---|---|---|
| PII 마스킹 | `maskPII(text)` | 신호 발췌(excerpt)에서 전화번호·계좌번호·주민번호·카드번호의 가운데를 `****`로 치환해 결과 화면에 원문 노출 방지 |
| 입력 검사 | `guardInput(text)` | 빈/공백 입력은 거부 메시지 반환(ok:false), 5000자 초과 입력은 앞부분만 잘라 분석(ok:true, note 포함) |
| 출력 검증 | `guardOutput(result)` | "100% 사기" 같은 과도한 단정 표현을 완화하고, "최종 판단은 사용자 몫이며, 의심되면 공식 기관에 직접 확인하세요." 면책 문구를 항상 추가 |
| 인젝션 탐지 | `detectPromptInjection(text)` | "이전 지시 무시", "시스템 프롬프트", "ignore previous" 등 프롬프트 인젝션 의심 패턴을 규칙 기반으로 탐지해 경고 출력. V2 LLM 도입 대비 입력단 방어 계층 |

## DAY13 반복 개선 — 추천 행동(actionAdvice) 추가

DAY12 중간 데모 피드백 #1("위험은 알겠는데 그래서 어떻게 하라는 건지 행동 안내가 약하다") 반영.

`ScamResult`에 `actionAdvice: string` 필드가 추가되어 위험 등급별 즉시 취할 행동을 안내합니다. 분류 로직(점수·등급)은 변경하지 않고 출력 전용으로만 동작합니다.

| 등급 | 추천 행동 |
|---|---|
| 매우위험 / 위험 | 응답·송금·링크 클릭을 멈추고, 기관/지인은 공식 번호로 직접 확인하세요. 이미 입력했다면 즉시 차단·신고(112/118)하세요. |
| 주의 | 바로 응하지 말고 출처를 한 번 더 확인하세요. 모르는 링크·계좌는 누르거나 보내지 마세요. |
| 안전 | 특이 신호는 없지만, 모르는 사람의 송금·링크 요구는 항상 의심하세요. |

CLI 출력에 "추천 행동  :" 줄이 추가됩니다.

WAF(웹 방화벽) 설정은 배포 단계(Cloudflare)의 책임 범위이며 이 코드에 포함하지 않습니다.
